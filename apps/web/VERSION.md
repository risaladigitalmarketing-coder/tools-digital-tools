# Version

Current version: **1.0.0**

### Semantic versioning rules (MAJOR.MINOR.PATCH)

- **MAJOR** – incompatible API changes or a complete redesign. Increment when you break existing tooling, remove public pages, or change the URL structure.
- **MINOR** – add backwards‑compatible functionality (new tools, pages, SEO enhancements). Increment when you add a new feature that does not break existing usage.
- **PATCH** – bug‑fixes, performance tweaks, documentation updates. Increment for any change that is fully backwards compatible.

**When to bump:**
- After a successful release that includes new tools → MINOR.
- After fixing a broken link or typo → PATCH.
- After changing the navigation schema or removing a tool → MAJOR.
